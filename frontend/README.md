# КАДРЫ — Фронтенд (Next.js)

Веб-интерфейс мультитенантной HR-системы холдинга.

## Стек

- **Next.js 15** + React 19 + TypeScript
- **Tailwind CSS v4** + shadcn/ui
- **TanStack React Table** — таблицы с пагинацией и фиксированными колонками
- **XLSX (SheetJS)** — экспорт в Excel
- **lucide-react** — иконки

## Запуск (Docker)

```bash
# Из корня проекта
docker compose up -d --build frontend

# Просмотр логов
docker compose logs -f frontend
```

## Запуск локально

```bash
npm install
cp .env.local.example .env.local   # настроить NEXT_PUBLIC_API_URL
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000)

## Структура страниц

```
app/
├── (auth)/
│   └── login/              # Страница входа
├── (app)/                  # Защищённые маршруты (требуют JWT)
│   ├── dashboard/          # Главная — статистика, быстрые действия
│   ├── employees/          # Список сотрудников
│   │   └── [id]/           # Профиль сотрудника (вкладки)
│   ├── departments/        # Отделы
│   ├── positions/          # Должности
│   ├── attendance/         # Посещаемость с цветовым кодированием
│   ├── org-structure/      # Оргструктура: дерево + по отделам
│   ├── reports/            # Отчёты (экспорт Excel)
│   ├── settings/           # Настройки аккаунта и расписания компании
│   ├── admin/              # Управление пользователями (суперадмин)
│   ├── doors/              # Управление дверями СКУД (суперадмин)
│   ├── registrations/      # QR-регистрация: токены + заявки + одобрение
│   ├── qr/                 # QR-чекин — отображение QR для входа/выхода
│   ├── salary/             # (скрыто) Зарплатные ведомости
│   └── inventory/          # (скрыто) Инвентарь
├── checkin/
│   └── phone/              # Публичный мобильный чекин (телефон + селфи)
└── register/               # Публичная форма регистрации по QR-коду
```

### Публичные страницы (не требуют входа)

| Страница | Путь | Описание |
|---|---|---|
| Вход | `/login` | Форма авторизации |
| QR-регистрация | `/register?token=...` | Самостоятельная регистрация сотрудника |
| Мобильный чекин | `/checkin/phone` | Отметка по номеру телефона + селфи |

### Скрытые модули

Зарплата (`/salary`) и Инвентарь (`/inventory`) скрыты в меню по решению руководства.
Код сохранён. Чтобы включить — раскомментировать соответствующие строки в `components/dashboard-layout.tsx`.

## Ключевые компоненты

| Компонент | Описание |
|---|---|
| `AuthContext` | Авторизация, роль, компания, переключение компаний для суперадмина |
| `CompanySelector` | Выбор компании в сайдбаре (только суперадмин) |
| `EmployeeAvatar` | Аватар с JWT-авторизацией для защищённых фото |
| `DataTable` | TanStack Table — таблицы с пагинацией, поиском, фикс. колонками |
| `PhotoCapture` | Камера с силуэтом для регистрации и мобильного чекина |
| `PhotoLightbox` | Просмотр фото в полном размере |
| `StatusBadge` | Переиспользуемый бейдж статуса сотрудника |

## API клиент

- `lib/hrms-api.ts` — все функции для работы с API (сотрудники, посещаемость, чекин и т.д.)
- `lib/types.ts` — TypeScript типы
- `lib/utils.ts` — вспомогательные функции

Все защищённые запросы добавляют JWT заголовок автоматически.
Компания передаётся через `withCompanyId()` — читает `currentCompanyId` из localStorage.

## Роли и доступ

| Роль | Доступ |
|---|---|
| **Суперадмин** | Все компании, переключатель компаний, страница СКУД/двери, админ-панель |
| **Кадровик** | Полный CRUD своей компании, настройки расписания, регистрации |
| **Руководитель** | Просмотр сотрудников + корректировка посещаемости |
| **Бухгалтер** | Только просмотр профилей сотрудников |

## Переменные окружения

| Переменная | Описание |
|---|---|
| `NEXT_PUBLIC_API_URL` | URL бэкенда (например, `/api` через Nginx или `http://localhost:7272/api`) |

## Особенности реализации

- **Мультитенантность**: `currentCompanyId` в `localStorage`, передаётся в каждый запрос через `withCompanyId()`
- **Фото сотрудников**: требуют JWT-авторизации — используйте `EmployeeAvatar`, не `<img>` напрямую
- **Dynamic Tailwind классы**: не работают с JIT (напр. `` `text-${color}-600` ``) — использовать явные классы
- **Next.js 15 params**: в client-компонентах использовать `React.use(params)` (params — Promise)
- **Обед**: при отображении отработанного времени учитывается настройка компании (12:00–13:00 по умолчанию)
- **Мобильный чекин**: защищён rate limiting (5 запр/мин), cooldown 5 мин, фото обязательно
