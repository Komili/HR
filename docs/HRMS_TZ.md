# HRMS — Техническое задание (актуальная версия)

## Назначение системы

Мультитенантная веб-система управления персоналом для холдинга из 8 компаний. Русскоязычный интерфейс. Работа в локальной сети.

---

## Технологический стек

| Компонент | Технология |
|---|---|
| Бэкенд | NestJS 11 + TypeScript + Prisma ORM |
| Фронтенд | Next.js 15 + React 19 + Tailwind CSS + shadcn/ui |
| База данных | MySQL 8.0 |
| Авторизация | JWT + Passport.js + bcrypt |
| Контейнеризация | Docker + docker-compose + Nginx |
| Таблицы | TanStack React Table |
| Экспорт | XLSX (SheetJS) |

---

## Архитектура (мультитенантность)

- Единая база данных, все таблицы содержат `companyId`
- Суперадмин холдинга видит все компании и может переключаться между ними
- Обычные пользователи видят только данные своей компании
- Изоляция данных обеспечивается на уровне бэкенда (`ForbiddenException` при нарушении)

---

## Роли (RBAC)

| Роль | Права |
|---|---|
| Суперадмин | Полный доступ ко всем компаниям, управление пользователями и компаниями |
| Кадровик | Полный CRUD в своей компании (сотрудники, отделы, должности, документы, инвентарь, посещаемость, настройки расписания) |
| Руководитель | Просмотр + корректировка посещаемости |
| Бухгалтер | Только просмотр профилей сотрудников |
| Сотрудник | Нет доступа к данным системы |

---

## Реализованные модули

### 1. Авторизация
- Вход по email и паролю
- JWT токен с полями: `email`, `sub`, `role`, `companyId`, `isHoldingAdmin`
- Автоматический выход при истечении токена

### 2. Дашборд
- Статистика: количество сотрудников, отделов, должностей, инвентаря
- Быстрые действия по роли
- Для суперадмина: сводка по всем компаниям холдинга

### 3. Сотрудники
- Список с пагинацией, поиском, фильтрацией по отделу/должности/статусу
- Профиль: личные данные, паспортные данные, рабочие данные, фото
- Индикатор заполненности профиля (%)
- Статусы: Активен / В отпуске / Уволен
- Вкладки профиля: Основные данные, Документы, Инвентарь, Посещаемость, История должностей

### 4. Документы сотрудника
- Загрузка файлов по типам (Паспорт, ИНН, Трудовой договор и др.)
- Хранение: `storage/companies/{Компания}/employees/{Имя_Фамилия_ID}/docs/`
- Скачивание и просмотр через API с JWT-авторизацией

### 5. Отделы и Должности
- CRUD с реальным подсчётом сотрудников
- Уникальные в рамках компании

### 6. Инвентарь
- CRUD с категориями, инвентарными номерами, ценами
- Закрепление инвентаря за сотрудниками
- История изменений (аудит)

### 7. Посещаемость
- Регистрация событий IN/OUT (СКУД Hikvision)
- Автоматический пересчёт дня при каждом событии
- **Вычет перерыва на обед** из отработанного времени
- Цветовое кодирование статусов (присутствует, ушёл, отсутствует)
- Корректировка времени (±минуты) с примечанием
- Экспорт в Excel

### 8. Зарплата
- Расчёт на основе посещаемости: `(оклад / рабочих_дней) × отработанных_дней`
- Ручные надбавки и удержания с примечанием
- Зарплатная ведомость с экспортом в Excel

### 9. Отчёты
- Экспорт в Excel: список сотрудников, посещаемость за период, месячный обзор

### 10. Настройки
- Смена пароля
- **Расписание обеда компании** (начало и конец перерыва, влияет на расчёт посещаемости)

### 11. История должностей
- Хронологическая запись изменений должности/отдела сотрудника
- Хранит снапшот названий (не теряется при удалении отдела/должности)

### 12. Регистрация через QR-код
- Кадровик генерирует QR-код со ссылкой-токеном
- Сотрудник сканирует QR и заполняет форму с фото через камеру
- Кадровик одобряет или отклоняет заявку
- При одобрении — создаётся профиль сотрудника

### 13. Управление пользователями (суперадмин)
- CRUD пользователей всех компаний
- Назначение ролей и компаний

---

## Расписание обеда

Каждая компания хранит время перерыва (`lunchBreakStart`, `lunchBreakEnd`, по умолчанию 12:00–13:00).

При регистрации события IN/OUT система автоматически вычитает пересечение рабочего времени с обеденным перерывом из `totalMinutes`.

Настраивается через страницу «Настройки» (роль: Кадровик, Руководитель, Суперадмин).

## Docker команды

```bash
# Запуск всех сервисов
docker-compose up -d --build

# Первичное заполнение БД
docker-compose exec backend node prisma/seed.js

# Применить изменения схемы
docker-compose exec backend npx prisma db push

# Логи
docker-compose logs -f backend
docker-compose logs -f frontend
```
