# Контекст проекта GEMINI

## Обзор каталога

Этот каталог содержит документы с техническим заданием для системы управления персоналом (HRMS). Цель проекта — создание безопасного веб-приложения для отдела кадров, предназначенного для управления данными и документами сотрудников в локальной сети компании. В спецификациях подробно описаны архитектура, функционал, роли пользователей и стратегия развертывания системы.

## Ключевые файлы

*   **`ТехЗадание.md`**: Основной и наиболее подробный документ с техническим заданием. Он описывает полный объем проекта, включая:
    *   Функциональные требования (профили сотрудников, управление документами, отчеты).
    *   Модель контроля доступа на основе ролей (RBAC): Кадровик, Руководитель, Бухгалтер, Сотрудник.
    *   Рекомендуемый современный технологический стек: **NestJS** (бэкенд), **Next.js** (фронтенд), **MySQL** (база данных) и **Tailwind CSS**.
    *   Конкретную структуру проекта для фронтенда и бэкенда.
    *   Полную конфигурацию `docker-compose.yml` для изолированного развертывания.
    *   Правила хранения файлов и соглашения об именовании для документов сотрудников.

*   **`HRMS_TZ.md`**: Этот файл представляет собой краткое изложение технического задания. Он охватывает основные функциональные требования, роли пользователей и ключевую цель приложения HRMS, служа сжатым обзором проекта.

## Использование

Эти документы являются основой для разработки приложения HRMS. Они предназначены для использования разработчиками, менеджерами проектов и другими заинтересованными сторонами для:
*   Понимания функционала и масштаба проекта.
*   Руководства при технической реализации и проектировании архитектуры.
*   Настройки окружений для разработки и эксплуатации с использованием предоставленной конфигурации Docker.
