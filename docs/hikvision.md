## 1. Как это работает

Интеграция состоит из двух независимых потоков:

```
┌─────────────────────────────────────────────────────────────────┐
│                     СЕРВЕР (интернет)                            │
│                                                                   │
│   Кадровик → "Выдать доступ" → HikvisionCommand (pending)       │
│   Webhook → AttendanceEvent → Telegram уведомление               │
└────────┬────────────────────────────────────┬────────────────────┘
         │                                    │
         │ Webhook (события прохода)           │ Polling команд
         │ POST /api/hikvision/event           │ GET /api/agent/hik-commands
         │ порт 7272                           │ каждые 5 сек
         ▼                                    ▼
┌──────────────────┐              ┌───────────────────────┐
│ Hikvision        │◀─────────────│  Relay Agent          │
│ DS-K1T344MBFWX   │  HTTP ISAPI  │  (ПК в офисе)         │
│ 192.168.0.199    │              │                       │
└──────────────────┘              └───────────────────────┘
```

| Поток | Направление | Назначение |
|-------|-------------|------------|
| **Webhook** | Устройство → Сервер | Фиксировать проходы сотрудников |
| **Relay Agent** | Сервер → Агент → Устройство | Добавлять / удалять Face ID |

## 2. Вход в веб-интерфейс устройства

1. Узнай IP-адрес устройства (отображается на экране терминала)
2. Открой браузер: `http://192.168.0.199`
3. Войди:
   - Логин: `admin`
   - Пароль: пароль установленный при первичной настройке

Меню слева:
```
⚙ (Настройки)
  ├── Система
  │   └── Системные настройки   ← информация об устройстве, время
  ├── Сеть
  │   ├── Сетевые настройки     ← Wi-Fi, IP адрес
  │   ├── Сетевая служба        ← Webhook (Мониторинг HTTP)
  │   └── Доступ к устройству   ← ISUP, Hik-Connect
  ├── Конфигурация доступа
  ├── Событие
  └── Посещаемость платформы
```

### Информация об устройстве

**Система → Системные настройки → вкладка "Базовая информация"**

Здесь видны: модель, серийный номер, версия прошивки, ёмкость (сколько лиц/карт записано).

## 3. Настройка сети Wi-Fi

**Сеть → Сетевые настройки → вкладка "Wi-Fi"**

### Подключение к сети

1. Переключатель **Wi-Fi** — включён (зелёный)
2. В таблице "Список сетей Wi-Fi" нажми **Подключение** напротив нужной сети
3. Введи пароль → статус станет **Подключено**

### Статический IP (обязательно)

В секции **WLAN** внизу страницы — отключи **DHCP** и заполни:

| Поле | Значение |
|------|----------|
| IPv4-адрес устройства | `192.168.0.199` |
| Маска подсети IPv4 | `255.255.255.0` |
| Шлюз по умолчанию IPv4 | `192.168.0.1` |
| Предпочтительный DNS-сервер | `192.168.0.1` |

Нажми **Сохранить**.

> Статический IP обязателен — relay-агент подключается к устройству по этому адресу. Если IP меняется при перезагрузке — агент не сможет записать Face ID.

## 4. Настройка Webhook — события прохода на сервер

**Сеть → Сетевая служба → вкладка "HTTP(S)"**

Страница разделена на три секции: **HTTP**, **HTTPS**, **Мониторинг HTTP**.

Настраивать нужно только секцию **Мониторинг HTTP**:

| Поле | Значение |
|------|----------|
| IP-адрес/доменное имя тревоги по событию | `185.125.200.112` |
| URL | `/api/hikvision/event?token=ВАШ_ТОКЕН` |
| Номер порта | `7272` |
| Протокол | `HTTP` (радиокнопка) |

Нажми **Сохранить**.

> **Токен** берётся из `backend/.env` на сервере, поле `HIKVISION_WEBHOOK_TOKEN`.  
> Пример URL: `/api/hikvision/event?token=a1b2c3d4e5f67890`

> **Про порт:** `7272` — прямой доступ к backend. `7474` — через Nginx. Для webhook используй `7272`.

### Проверка

После сохранения устройство начнёт отправлять heartbeat каждые ~30 секунд. Зайди в КАДРЫ как суперадмин → **Управление дверями** → вкладка **Устройства Hikvision** — устройство появится со статусом `pending`.

## 5. Relay Agent — выдача доступа через локальный ПК

Relay Agent — скрипт `agent/relay-agent.js`, который запускается на **ПК в локальной сети офиса**. Сервер не может напрямую достучаться до Hikvision (устройство за NAT), поэтому агент сам опрашивает сервер каждые 5 сек и выполняет команды.

### Требования

- ПК в той же Wi-Fi/LAN сети что и Hikvision устройство
- Node.js 18+ — скачать на [nodejs.org](https://nodejs.org) (кнопка LTS)
- Постоянный интернет на ПК

### Установка (Windows) — один двойной клик

1. Скопировать папку `agent/` на ПК
2. Двойной клик на **`AGENT.bat`**
3. Скрипт автоматически запросит права администратора и всё сделает сам:
   - Проверит Node.js
   - Скопирует файлы в `C:\Hikvision\`
   - Запишет `config.json` с настройками сервера
   - Зарегистрирует задачу в Планировщик Windows (автозапуск при старте, перезапуск при сбое)
   - Запустит агент немедленно
   - Создаст ярлык **"HRMS Agent Logs"** на рабочем столе

Настройки захардкожены в `AGENT.bat` (менять при смене сервера):

| Параметр | Значение |
|----------|----------|
| `serverUrl` | `http://185.125.200.112:7272` |
| `agentToken` | `AGENT_SECRET_TOKEN` из `backend/.env` |
| `companyId` | `1` (Фавз) |

**Повторный клик** на `AGENT.bat` — показывает меню:
- Enter → перезапустить
- U → удалить агент
- R → переустановить (обновить файлы)

> ⚠️ **Токен агента ≠ Токен Hikvision-вебхука!**
> `HIKVISION_WEBHOOK_TOKEN` — вставляется в настройки устройства (URL событий).
> `AGENT_SECRET_TOKEN` — используется relay-агентом, прописан в `AGENT.bat`.

### Автозапуск — Linux

```bash
sudo nano /etc/systemd/system/hrms-agent.service
```

```ini
[Unit]
Description=HRMS Relay Agent
After=network-online.target

[Service]
WorkingDirectory=/opt/hrms-agent
ExecStart=/usr/bin/node relay-agent.js
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable --now hrms-agent
```

### Проверка работы

Нормальный вывод в логах (`agent.log`):
```
[INFO ] HRMS Relay Agent v1.0
[INFO ] Сервер: http://185.125.200.112:7272
[OK   ] 🟢 Сервер доступен. Команд в очереди: 0
[INFO ] 📡 Офис (IN) 192.168.0.199 → 🟢 онлайн
```

## 6. Привязка устройства в системе КАДРЫ

После того как устройство появилось в системе (статус `pending`):

1. Войди как **суперадмин**
2. **Управление дверями** → вкладка **Устройства Hikvision**
3. Найди устройство → нажми **Привязать**
4. Заполни:

| Поле | Значение |
|------|----------|
| Компания | Компания этого офиса |
| Офис | Название (например: `Главный офис`) |
| Направление | `IN` — вход, `OUT` — выход |
| Логин | `admin` |
| Пароль | Пароль от веб-интерфейса устройства |

5. Сохрани → статус станет `active`

## 7. Выдача доступа сотрудникам

### Как выдать

1. Профиль сотрудника → вкладка **Доступ**
2. Выбери устройство → **Выдать доступ**

### Что происходит (цепочка)

```
Кнопка нажата
  ↓
Сервер создаёт HikvisionCommand { action: grant, status: pending }
  ↓ (через 0-5 сек)
Relay Agent забирает команду
  ↓
Агент → DELETE старого пользователя (чтоб не было дубля)
Агент → POST добавить пользователя (employeeNo = ID сотрудника)
  ↓
Агент скачивает фото сотрудника с сервера
Агент → POST загрузить фото Face ID (JPEG, max 640×640)
  ↓
Агент → сервер: { status: done }
  ↓
Telegram: ✅ Face ID добавлен на устройство
```

### Лог агента при успехе

```
[INFO ] 📥 Hik Face ID команд: 1
[INFO ] ▶ Hik-команда #42 [GRANT] Иванов Иван → "Главный офис (Вход)" (192.168.0.199)
[OK   ]   ✓ Пользователь добавлен на 192.168.0.199
[OK   ]   ✓ Face ID загружен (45 KB)
[OK   ] ✅ Hik-команда #42 выполнена
```

### Статусы команды

| Статус | Значение |
|--------|----------|
| `pending` | Ожидает агента |
| `processing` | Агент выполняет |
| `done` | Успешно |
| `failed` | Ошибка (см. поле `error`) |

## 8. Как фиксируется посещаемость

```
Сотрудник прикладывает лицо → устройство открывает дверь
  ↓
Hikvision → POST /api/hikvision/event (webhook)
  ↓
Сервер ищет сотрудника:
  WHERE skudId = employeeNo OR id = employeeNo
  (поддерживает и старый СКУД, и новых сотрудников)
  ↓
Создаёт AttendanceEvent { direction: IN/OUT, source: HIKVISION }
  ↓
Пересчёт дневного резюме посещаемости
  ↓
Telegram уведомление
```

### Когда событие игнорируется

- Устройство не привязано к компании (статус `pending`)
- Нет `employeeNo` в событии (heartbeat, системные события)
- Сотрудник не найден в БД

## 10. Настройка ISUP (необязательно)

> ISUP не нужен если используется relay-агент. Relay-агент — основной и надёжный способ.

ISUP позволяет устройству самому подключаться к серверу через TCP. Используется если нет возможности держать relay-агент.

**Сеть → Доступ к устройству → вкладка "ISUP"**

| Поле | Значение |
|------|----------|
| Включить | вкл |
| Версия протокола | ISUP5.0 |
| IP-адрес сервера | `185.125.200.112` |
| Номер порта | `7660` |
| Идентификатор устройства | серийный номер устройства |
| Ключ шифрования | оставь пустым |

После сохранения нажми **Обновить** рядом со "Статус регистрации" — должно появиться **В сети**.

## 11. Переменные окружения

В файле `backend/.env` на сервере:

```env
# Токен для webhook (добавляется в URL устройства: ?token=...)
HIKVISION_WEBHOOK_TOKEN=случайная_строка

# Токен для relay-агента (указывается в config.json агента)
AGENT_SECRET_TOKEN=другая_случайная_строка

# TCP порт для ISUP (устройства подключаются сами)
ISUP_PORT=7660
ISUP_ENC_KEY=       # ключ шифрования EHome2.0 (обычно пусто)
```

Сгенерировать токен:
```bash
node -e "console.log(require('crypto').randomBytes(16).toString('hex'))"
```

## 12. Устранение неполадок

### Устройство не появляется в системе КАДРЫ

- Webhook не настроен или настроен неправильно
- Проверь поля: IP сервера `185.125.200.112`, порт `7272`, URL с токеном
- Устройство должно иметь доступ в интернет (Wi-Fi подключён)
- Нажми **Сохранить** в настройках Мониторинг HTTP ещё раз

### Событие прохода не записывается в посещаемость

- Устройство должно быть привязано (статус `active`, не `pending`)
- У сотрудника должно совпадать `id` или `skudId` с `employeeNo` на устройстве
- Логи backend: `docker compose logs backend | grep hikvision`

### Relay Agent: фото Face ID не загружается

| Ошибка устройства | Причина | Решение |
|-------------------|---------|---------|
| `SubpicAnalysisModelingError` | Лицо не найдено на фото | Заменить фото (чёткое, анфас, хорошее освещение) |
| `FaceDataRecordCapacityReached` | Память устройства заполнена | Удалить старые лица через веб-интерфейс |
| `Timeout 192.168.x.x` | Устройство недоступно | Проверить Wi-Fi, пинг |
| `HTTP 401` | Неверный пароль устройства | Исправить пароль в привязке устройства в UI |

### Команды зависли в статусе `processing`

Агент взял команду и упал. Перезапусти агент — команды сбросятся в `pending` автоматически.

### Сбросить зависшие команды вручную (SQL)

Если агент упал и команды зависли в `processing` — сбросить их в `pending`:

```sql
UPDATE HikvisionCommand
SET status = 'pending'
WHERE status = 'processing'
  AND updatedAt < DATE_SUB(NOW(), INTERVAL 10 MINUTE);
```

Посмотреть все незавершённые команды:

```sql
SELECT hc.id, hc.action, hc.status, hc.error, hc.createdAt,
       hd.officeName, hd.direction,
       CONCAT(e.lastName, ' ', e.firstName) AS employee
FROM HikvisionCommand hc
JOIN HikvisionDevice hd ON hd.id = hc.deviceId
JOIN Employee e ON e.id = hc.employeeId
WHERE hc.status != 'done'
ORDER BY hc.createdAt DESC;
```

### Relay Agent не видит сервер

```
[ERROR] Не могу достучаться до сервера
```

- Проверь интернет на ПК с агентом
- Проверь `serverUrl` в `config.json`
- Проверь что `agentToken` совпадает с `AGENT_SECRET_TOKEN` в `.env`
