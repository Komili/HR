# Руководство по эксплуатации и использованию системы HRMS

## 1. Введение

Данное руководство предназначено для двух типов пользователей:
*   **Администраторы системы**: Ответственные за установку, запуск, обслуживание и резервное копирование системы.
*   **Конечные пользователи**: Сотрудники отдела кадров, бухгалтерии и руководители, которые будут работать с данными в системе.

---

## 2. Руководство по эксплуатации (для администраторов)

Этот раздел описывает технические аспекты развертывания и поддержки системы.

### 2.1. Требования к системе
*   **Сервер**: Любой сервер (физический или виртуальный) с установленным **Docker** и **Docker Compose**.
*   **Git**: Для получения исходного кода проекта.

### 2.2. Первый запуск и установка
1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://github.com/Komili/HR.git
    cd HR
    ```
2.  **Настройте переменные окружения**:
    Создайте файл `.env` в корневой папке проекта, скопировав содержимое из `.env.example` (если он есть) или создав его с нуля. Заполните его надежными паролями:
    ```ini
    # Переменные для базы данных
    MYSQL_PASSWORD=ВАШ_НАДЕЖНЫЙ_ПАРОЛЬ_ПОЛЬЗОВАТЕЛЯ
    MYSQL_ROOT_PASSWORD=ВАШ_НАДЕЖНЫЙ_ПАРОЛЬ_ROOT
    DATABASE_URL="mysql://hrms:ВАШ_НАДЕЖНЫЙ_ПАРОЛЬ_ПОЛЬЗОВАТЕЛЯ@hrms_db:3306/hrms"
    ```
    **Внимание**: Пароль в `MYSQL_PASSWORD` и `DATABASE_URL` должен совпадать.

3.  **Запустите проект**:
    Выполните команду в корневой папке проекта:
    ```bash
    docker-compose up --build -d
    ```
    Эта команда соберет Docker-образы для бэкенда и фронтенда, скачает образы MySQL и Nginx и запустит все контейнеры в фоновом режиме. Первый запуск может занять 5-15 минут.

### 2.3. Повседневное управление
*   **Чтобы запустить остановленные контейнеры**:
    ```bash
    docker-compose start
    ```
*   **Чтобы остановить контейнеры (без удаления данных)**:
    ```bash
    docker-compose stop
    ```
*   **Чтобы полностью остановить и удалить контейнеры**:
    ```bash
    docker-compose down
    ```
    (Данные в базе и загруженные файлы при этом сохранятся, так как они находятся в Docker-томах (volumes)).

### 2.4. Резервное копирование
Крайне важно регулярно создавать резервные копии данных. В системе есть два типа данных, которые нужно бэкапить:

1.  **База данных MySQL**: Хранится в Docker-томе `hrms_mysql`.
2.  **Загруженные файлы сотрудников**: Хранятся в Docker-томе `hrms_storage`.

Для создания резервной копии необходимо использовать стандартные средства `docker` для копирования данных из томов или специализированные утилиты для бэкапа баз данных (например, `mysqldump`).

### 2.5. Обновление системы
1.  **Получите последнюю версию кода**:
    ```bash
    git pull
    ```
2.  **Пересоберите образы и перезапустите контейнеры**:
    ```bash
    docker-compose up --build -d
    ```

## 3. Руководство пользователя

Этот раздел описывает, как работать с функциями веб-приложения.

### 3.1. Доступ к системе
Откройте ваш веб-браузер и перейдите по адресу, предоставленному системным администратором (по умолчанию `http://localhost:8088`, если вы запускаете систему на своем компьютере).

### 3.2. Вход в систему
Для входа используйте логин и пароль, выданные вам администратором. В зависимости от вашей роли (Кадровик, Руководитель и т.д.), вам будут доступны разные разделы и функции.

### 3.3. Основные разделы

#### 3.3.1. Дашборд (Главная страница)
После входа вы попадаете на Дашборд, где отображается общая статистика:
*   Общее количество сотрудников.
*   Количество новых сотрудников за последний месяц.
*   Список сотрудников, чей профиль требует заполнения.

#### 3.3.2. Сотрудники
Это основной раздел для работы с персоналом.
*   **Просмотр списка**: Здесь отображается таблица со всеми сотрудниками. Вы можете использовать поиск и фильтры для быстрого нахождения нужного человека.
*   **Создание нового сотрудника**:
    1.  Нажмите кнопку "Добавить сотрудника".
    2.  Заполните все необходимые поля в форме, включая имя и фамилию на латинице (это важно для создания папки для документов).
    3.  Нажмите "Сохранить".
*   **Редактирование профиля**:
    1.  Найдите сотрудника в списке и кликните по его имени, чтобы перейти в его профиль.
    2.  Нажмите кнопку "Редактировать" и внесите необходимые изменения.

#### 3.3.3. Управление документами
Загрузка и управление документами производятся непосредственно из профиля сотрудника.
1.  Откройте профиль нужного сотрудника.
2.  Перейдите на вкладку "Документы".
3.  Вы увидите список из 9 обязательных типов документов.
4.  Напротив нужного типа документа нажмите кнопку "Загрузить".
5.  Выберите файл на вашем компьютере.
6.  После загрузки статус документа изменится, и вы сможете его посмотреть, скачать или заменить.
