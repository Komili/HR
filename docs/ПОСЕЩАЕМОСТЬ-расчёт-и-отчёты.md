# Посещаемость: как считается время и какие отчёты формирует система

> Основано на коде: `backend/src/attendance/attendance.service.ts`,
> `frontend/app/(app)/dashboard/page.tsx`, `frontend/app/(app)/reports/page.tsx`.

---

## 1. Как считается отработанное время за один день

Система смотрит только на **самое раннее IN** и **самое позднее OUT** за календарный
день. Все события между ними (например, ушёл на обед не по расписанию, вышел на
5 минут) на расчёт **не влияют** — они просто не участвуют в формуле.

```
00:00                                                              23:59
  │                                                                  │
  │        08:55        12:00        13:00        18:10             │
  │          │             │            │            │               │
  │         IN         OUT (обед)   IN (обед)       OUT              │
  │          │             │            │            │               │
  │      firstEntry ─────────────────────────────── lastExit         │
  │          └──────────────── totalMinutes ──────────────┘          │
  │                      минус пересечение с обедом                  │
```

**Формула** (`recalculateDay`, attendance.service.ts:821):

```
totalMinutes = (lastExit − firstEntry)  −  пересечение с обеденным перерывом компании
totalMinutes = totalMinutes + сумма минут из корректировок «±минуты» (только модуль, всегда прибавляет)
totalMinutes = max(0, totalMinutes)
```

Если IN есть, а OUT ещё нет (сотрудник ещё на месте) — `totalMinutes` не считается,
статус = `present`. Если событий за день вообще нет — записи `Attendance` не будет,
статус в выдаче = `absent`.

### Статус дня

| Статус | Условие |
|---|---|
| `present` | последнее событие дня — IN (или единственное событие — IN) |
| `left` | последнее событие дня — OUT |
| `absent` | за день нет ни одного события |
| `excused` | день вручную помечен через «Отпросился, ушёл / не пришёл» |

### Опоздание / ранний уход (на странице «Посещаемость» за один день)

```
isLate       = firstEntry > workDayStart      (компании)   — БЕЗ допуска, ровно по минутам
isEarlyLeave = lastExit   < workDayEnd        (компании)   — БЕЗ допуска, ровно по минутам
```

⚠️ Это важный нюанс: на дашборде бейдж «Опоздал» загорается уже на 1 минуту
просрочки. А в отчёте «Табель за период» (см. ниже) действует **допуск 15 минут** —
то есть один и тот же день может считаться «опозданием» на дашборде, но не
попасть в число опозданий в месячном табеле.

## 2. Как корректировки из диалога влияют на расчёт

Диалог «Что нужно сделать?» (кнопка ✏️) — 7 действий, каждое трогает данные
по-своему:

```
┌─────────────────────────────┬──────────────────┬──────────────────┬─────────────────────┐
│ Действие                    │ Меняет totalMinutes│ Создаёт события │ Меняет статус дня    │
├─────────────────────────────┼──────────────────┼──────────────────┼─────────────────────┤
│ Работал вне офиса           │ +N мин (магнитуда)│ нет              │ нет                  │
│ Отлучался по делам          │ +N мин (магнитуда)│ нет              │ нет                  │
│ Отметить приход (Check-In)  │ пересчитывается   │ да, IN           │ нет напрямую         │
│ Отметить уход (Check-Out)   │ пересчитывается   │ да, OUT          │ нет напрямую         │
│ Отлучался / вне офиса       │ НЕТ (только заметка)│ нет            │ нет                  │
│ Отпросился, ушёл            │ по факту событий  │ да, OUT (если указано время) │ excused   │
│ Отпросился, не пришёл       │ 0 (весь день)     │ нет              │ excused              │
└─────────────────────────────┴──────────────────┴──────────────────┴─────────────────────┘
```

Важные детали:

- **«Работал вне офиса» / «Отлучался по делам»** — единственные два действия,
  которые прибавляют время к `totalMinutes`. Оба всегда **прибавляют**, разницы
  в математике нет — разница только в *причине*, которая пишется в
  `correctionType` и знаком в `correctionMinutes` (используется только для
  индикатора в таблице: 🏗 offsite показывается как «−», 🚶 excused — как «+»).
  Реального вычитания времени в системе не существует в принципе — временные
  отлучки (ушёл на 2 часа и вернулся) **не уменьшают** отработанное время, только
  фиксируются заметкой (см. пункт «Отлучался / вне офиса»).

- **Check-In / Check-Out** создают настоящее событие `AttendanceEvent`, после
  чего день **пересчитывается заново** по формуле из раздела 1 — то есть если
  ручной Check-In оказался раньше текущего firstEntry, он станет новым
  firstEntry (и наоборот для Check-Out).

- **«Отлучался / вне офиса»** — просто пишет `correctionNote` (+опционально
  `correctionDeadline` — до какого времени ждём обратно). На `totalMinutes` и
  статус не влияет вообще.

- **«Отпросился, ушёл»** — если указано время ухода, создаёт реальное событие
  OUT и пересчитывает день (так что часы досчитываются как обычно до этого
  времени), затем весь день помечается статусом `excused`.

- **«Отпросился, не пришёл»** — создаёт запись `Attendance` с `totalMinutes = 0`,
  без событий, статус `excused`. В месячном отчёте такой день **не считается
  прогулом** (см. раздел 3), но и не считается физическим присутствием.

## 3. Какие отчёты выдаёт система

В системе **5 разных экспортов**, они разбросаны по двум страницам и НЕ
дублируют друг друга полностью:

```
┌──────────────────────────────────────────────────────────────────────────┐
│  /dashboard  (Посещаемость за день)                                       │
│  ├─ 📄 Кнопка «Excel» рядом с датой → «Посещаемость_ДД.ММ.ГГГГ.xlsx»      │
│  └─ 📅 Кнопка «За период» → «Посещаемость_ОТ_ДО.xlsx» (цветной, IN/OUT)  │
│                                                                            │
│  /reports  (Отчёты)                                                       │
│  ├─ 🗂 «Табель за период» → «Табель_ОТ_ДО.xlsx» (2 листа) — ГЛАВНЫЙ отчёт │
│  ├─ 👥 Сотрудники → «Сотрудники_ДД.ММ.ГГГГ.xlsx»                          │
│  ├─ 🏢 Отделы → «Отделы_ДД.ММ.ГГГГ.xlsx»                                  │
│  ├─ 📊 Посещаемость (сегодня) → «Посещаемость_ДД.ММ.ГГГГ.xlsx»            │
│  ├─ 📈 Обзор → «Месячный_обзор_ДД.ММ.ГГГГ.xlsx»                           │
│  └─ 📦 «Экспортировать всё» → «HR_Отчёты_ДД.ММ.ГГГГ.xlsx» (4 листа сразу) │
└──────────────────────────────────────────────────────────────────────────┘
```

### 3.1 «Посещаемость за день» (dashboard, кнопка рядом с выбором даты)

Один лист, один день, по строке на сотрудника.

```
┌────┬──────────┬───────┬──────────┬───────┬───────┬───────┬─────┬─────┬───────────┬───────────────┬──────────┐
│ №  │  ФИО     │ Отдел │Должность │ Офис  │ Вход  │ Выход │Часы │ Мин │ Опоздание │ Корректировка │ Статус   │
├────┼──────────┼───────┼──────────┼───────┼───────┼───────┼─────┼─────┼───────────┼───────────────┼──────────┤
│ 1  │ Иванов И.│ IT    │Разработ. │ Офис1 │ 09:05 │ 18:02 │  8  │  57 │  +5м      │               │ Ушёл     │
│ 2  │ Петров П.│ HR    │Кадровик  │ Офис1 │  —    │  —    │  0  │   0 │           │               │ Отсутств.│
└────┴──────────┴───────┴──────────┴───────┴───────┴───────┴─────┴─────┴───────────┴───────────────┴──────────┘
ИТОГО: ...ч ...м │ Опоздали: N чел. │ Отсутствовали: N чел.
```

Внизу — три строки итогов (сумма часов, опоздания, прогулы за выбранный день).

### 3.2 «За период» (dashboard, кнопка-календарь) — цветная таблица вход/выход по дням

Каждый день периода — это **две колонки** (Вход/Выход), сотрудники — строки.
Формируется на клиенте (ExcelJS), с подсветкой: жёлтый = опоздание, оранжевый =
ранний уход, розовый = выходной, зелёный = переработка.

```
┌────┬──────────┬───────────────┬───────────────┬───────────────┬────────┬───────────┬─────────────┬──────────┐
│ №  │  ФИО     │  01.07 (Вт)   │  02.07 (Ср)   │  03.07 (Чт)   │ Всего  │ Опозданий │ Переработка │ Пропусков│
│    │          │ Вход │ Выход  │ Вход │ Выход  │ Вход │ Выход  │  часов │ дн./время │  дн./время  │   дней   │
├────┼──────────┼──────┼────────┼──────┼────────┼──────┼────────┼────────┼───────────┼─────────────┼──────────┤
│ 1  │ Иванов И.│09:05 │ 18:02  │09:00 │ 18:10  │  —   │   —    │16ч 57м │1 дн.(5м)  │ 1 дн.(10м)  │    1     │
└────┴──────────┴──────┴────────┴──────┴────────┴──────┴────────┴────────┴───────────┴─────────────┴──────────┘
```

Опоздание/переработка считаются **на клиенте**, с допуском 15 мин (опоздание) и
5 мин (переработка/ранний уход) — так же, как в п. 3.3, но это **отдельный,
независимый расчёт** (не переиспользует backend-эндпоинт отчёта).

### 3.3 «Табель за период» (reports, главный отчёт) — 2 листа, считает backend

Данные берёт `GET /attendance/report` (`getRangeReport`) — единственный отчёт,
где вся арифметика посчитана **на сервере**, а не на клиенте.

**Лист 1 — «Табель»** (17 колонок, по строке на сотрудника):

```
№ │ ФИО │ Отдел │ Должность │ Отработано │ Рабочих дней │ Присутств. │ Прогулы │
  Опозд.дней │ Опозд.минут │ Опозд.время │ Без входа │ Без выхода │
  Ранний.дней │ Ранний.время │ Переработка.дней │ Переработка.время
```

**Лист 2 — «По дням»** — плоский список: Дата, День недели, ФИО, Отдел, Вход,
Выход, Часы, Пометки (Опоздание / Ранний уход / Без входа / Без выхода).

Пороги на сервере (attendance.service.ts:278-279):

```
LATE_GRACE_MIN = 15 мин   → день считается «опозданием» только если пришёл позже чем на 15 мин
GRACE_MIN      = 5 мин    → «переработка»/«ранний уход» считается от 5 мин отклонения
```

Как считаются метрики по каждому сотруднику за период:

```
workingDays  = дни периода, которые являются рабочими днями компании (workDays)
presentDays  = дни, где ЕСТЬ запись Attendance (включая «отпросился»!)
absentDays   = рабочий день БЕЗ записи Attendance = прогул
missingInDays / missingOutDays = есть запись, но нет firstEntry / lastExit
lateDays/Minutes       = firstEntry позже начала дня больше чем на 15 мин
earlyLeaveDays/Minutes = lastExit раньше конца дня больше чем на 5 мин
overworkDays/Minutes   = пришёл раньше начала (>5м) ИЛИ ушёл позже конца (>5м),
                          а если день выходной и есть часы — вся смена = переработка
```

⚠️ **Нюанс:** «Отпросился, не пришёл» создаёт запись `Attendance` (даже без
событий), поэтому такой день попадает в `presentDays`, а НЕ в `absentDays` —
то есть в табеле он не считается прогулом. Формально «присутствие» здесь не
означает физическое присутствие, а означает «есть объяснение, почему нет
данных».

### 3.4 Простые справочные отчёты (reports, кнопки-карточки)

Не связаны с расчётом времени — прямые выгрузки списков:

| Отчёт | Источник | Колонки |
|---|---|---|
| Сотрудники | `GET /employees` (все, до 1000) | ID, Фамилия, Имя, Отчество, Email, Телефон, Отдел, Должность |
| Отделы | список отделов + подсчёт сотрудников на клиенте | ID, Название, Кол-во сотрудников |
| Посещаемость (сегодня) | `GET /attendance?date=сегодня` | Сотрудник, Отдел, Должность, Вход, Выход, Часы, Статус |
| Обзор | агрегаты на клиенте | Всего сотрудников / отделов / должностей, На месте сегодня, Дата отчёта |
| «Экспортировать всё» | объединяет 4 отчёта выше в один файл, 4 листа | — |

frontend/app/(app)/dashboard/page.tsx

frontend/app/(app)/reports/page.tsx
  ├─ generateTimesheetReport() — «Табель за период» (2 листа)
