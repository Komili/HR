# КАДРЫ — Система управления персоналом

Мультитенантная HR-система для холдинга из 9 компаний с интеграцией СКУД Hikvision Face ID.

> **Новичок?** Начни с [docs/ИНСТРУКЦИЯ.md](docs/ИНСТРУКЦИЯ.md) — там всё по шагам с объяснениями.

---

## Быстрый старт

```bash
# 1. Скачать
git clone https://github.com/Komili/HR.git
cd HR

# 2. Настроить (заменить пароли в файле .env!)
cp .env.example .env
nano .env

# 3. Запустить (5–15 минут в первый раз)
docker compose up --build -d

# 4. Создать схему БД
docker compose exec backend npx prisma db push

# 5. Заполнить базовыми данными (роли, компании, суперадмин)
docker compose exec backend node prisma/seed.js

# 6. Создать папку для временных файлов (обязательно!)
docker exec hrms_backend mkdir -p /app/storage/tmp

# 7. Открыть http://localhost:7474
# Логин: admin@holding.tj / password
```

Подробная инструкция: **[docs/ИНСТРУКЦИЯ.md](docs/ИНСТРУКЦИЯ.md)**
Развёртывание на сервере: **[DEPLOYMENT.md](DEPLOYMENT.md)**

## СКУД — Hikvision Face ID

Система поддерживает контроль доступа через устройства Hikvision Face ID (DS-K1T342MX-E1).

### Как работает

```
Hikvision устройство
        │
        ├── HTTP Webhook ──────────────────→ Сервер (:7474/api/hikvision/event)
        │                                         │
        │                                    Запись в БД
        │                                    Telegram уведомление
        │
        └── Relay-агент (ПК в офисе)
                │
                ├── Polling команд ──────→ Сервер (:7474/api/agent/hik-commands)
                └── ISAPI вызовы ───────→ Устройство (:80) — выдача/отзыв доступа
```

- Webhook — устройство само отправляет событие на сервер (событие входа/выхода)
- Relay-агент — лёгкий Node.js скрипт (`agent/relay-agent.js`) на ПК в офисе, опрашивает сервер каждые 5 сек

Подробнее: **[docs/ИНСТРУКЦИЯ.md → Раздел Face ID](docs/ИНСТРУКЦИЯ.md)**

## Мобильный чекин по телефону

Публичная страница `/checkin/phone` позволяет сотрудникам отмечаться без QR-кода:

1. Сотрудник открывает страницу (ссылка в меню «Чекин по телефону»)
2. Вводит свой номер телефона (992XXXXXXXXX)
3. Делает селфи через камеру
4. Система определяет: первый скан дня = **вход**, последующие = **выход**
5. В Telegram приходит уведомление с фото

**Защита:** rate limiting 5 запросов/мин на IP, cooldown 5 мин между отметками, фото обязательно.

## Telegram уведомления

Система отправляет уведомления при:
- Входе/выходе сотрудников (Face ID и мобильный чекин)
- Опоздании на работу
- Раннем уходе
- Ошибках выполнения команд СКУД
- Устройство офлайн / восстановлено
- Дедлайнах корректировок посещаемости

Настройка: `TELEGRAM_TOKEN` и `TELEGRAM_CHAT_IDS` в `.env`

## Порты

| Сервис | Внешний порт | Описание |
|--------|-------------|----------|
| **Nginx (главный)** | **7474** | Основной вход в систему |
| MySQL | 7171 | База данных (только внутри сервера) |
| Backend API | 7272 | REST API (только внутри сервера) |
| Frontend | 7373 | Веб-интерфейс (только внутри сервера) |
| ISUP/EHome TCP | 7660 | Прямое подключение Hikvision устройств |

> Наружу открывать только порт **7474** (и **7660** если используется ISUP).

## Полезные команды

```bash
# Пересборка бэкенда после изменений кода
docker compose up -d --build backend

# Пересборка фронтенда
docker compose up -d --build frontend

# Просмотр логов в реальном времени
docker compose logs -f backend
docker compose logs -f frontend

# Статус всех контейнеров
docker compose ps

# Резервная копия БД
docker compose exec db mysqldump -u hrms -p'ПАРОЛЬ' hrms > backup_$(date +%Y%m%d).sql

# Полный сброс (⚠️ удаляет ВСЕ данные)
docker compose down -v && docker compose up --build -d
docker compose exec backend npx prisma db push
docker compose exec backend node prisma/seed.js
```
