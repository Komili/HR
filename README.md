# КАДРЫ — Система управления персоналом

Мультитенантная HR-система для холдинга из 8 компаний с интеграцией СКУД Hikvision Face ID.

## Быстрый старт

```bash
# 1. Скачать
git clone https://github.com/Komili/HR.git
cd HR

# 2. Настроить
cp .env.example .env
# Отредактировать .env — сменить пароли и токены!

# 3. Запустить (5-15 минут в первый раз)
docker compose up --build -d

# 4. Создать схему БД
docker compose exec backend npx prisma db push

# 5. Заполнить тестовыми данными
docker compose exec backend node prisma/seed.js

# 6. Открыть http://localhost:7474
# Логин: admin@holding.tj / password
```

Подробная инструкция по развёртыванию: **[DEPLOYMENT.md](DEPLOYMENT.md)**

## Модули системы

| Модуль | Описание |
|--------|----------|
| **Сотрудники** | CRUD, профили с вкладками, фото, документы, история должностей |
| **Отделы / Должности** | Уникальные в рамках каждой компании |
| **Посещаемость** | Учёт времени, корректировки (±мин, ручной вход/выход), цветовая кодировка |
| **Зарплата** | Расчёт на основе посещаемости, надбавки и удержания |
| **Инвентарь** | Учёт имущества, привязка к сотрудникам, история изменений |
| **Оргструктура** | Визуальная иерархия (дерево + по отделам) |
| **Регистрации** | QR-коды для самостоятельной регистрации сотрудников |
| **СКУД (Face ID)** | Управление дверями Hikvision, выдача/отзыв доступа, мониторинг устройств |
| **Админ-панель** | Управление пользователями, компаниями, ролями |

## СКУД — Hikvision Face ID

Система поддерживает контроль доступа через устройства Hikvision Face ID (DS-K1T342MX-E1).

**Архитектура:**
```
HRMS Сервер  ←── HTTPS ──→  Relay Agent (в офисе)  ←── HTTP локал. ──→  Hikvision
```

В каждом офисе устанавливается **relay-агент** (`agent/`) — лёгкий Node.js скрипт без зависимостей.
Он опрашивает сервер каждые 5 сек, выполняет ISAPI-команды и мониторит доступность устройств.

- **[docs/СКУД.md](docs/СКУД.md)** — полная документация по СКУД
- **[agent/README.md](agent/README.md)** — установка relay-агента в офисе

## Telegram уведомления

Система отправляет уведомления при:
- Старте/остановке сервера
- Выдаче/отзыве доступа к дверям и подтверждении от агента
- Ошибках выполнения команд СКУД
- Устройство офлайн / восстановлено
- Дедлайнах корректировок посещаемости

Настройка: `TELEGRAM_TOKEN` и `TELEGRAM_CHAT_IDS` в `backend/.env`

## Полезные команды

```bash
# Пересборка после изменений бэкенда
docker compose up -d --build backend

# Пересборка фронтенда
docker compose up -d --build frontend

# Логи
docker compose logs -f backend
docker compose logs -f frontend

# Резервная копия БД
docker compose exec db mysqldump -u hrms -pПАРОЛЬ hrms > backup_$(date +%Y%m%d).sql

# Полный сброс
docker compose down -v && docker compose up --build -d
docker compose exec backend npx prisma db push
docker compose exec backend node prisma/seed.js
```
