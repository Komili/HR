# HRMS Relay Agent — Агент управления Hikvision Face ID

Лёгкий скрипт на Node.js, который запускается на Windows-компьютере в локальной сети офиса.
Без него невозможно выдавать и отзывать доступы сотрудников на Hikvision-терминалах.

---

## Как это работает

```
Сервер HRMS (интернет)
        │
        │  каждые 5 сек агент спрашивает:
        │  "есть ли команды для выполнения?"
        │
Relay Agent (ПК в офисе)
        │
        │  если есть команда — делает ISAPI запрос:
        ▼
Hikvision Face ID (192.168.x.x)  — выдаёт или отзывает доступ
```

Агент — это посредник. Сервер HRMS недоступен напрямую к устройствам (они в локальной сети офиса),
поэтому агент сидит в офисной сети и выполняет команды от имени сервера.

---

## Требования

- **Windows 10/11** (или Windows Server 2019+)
- **Node.js 18 LTS** или новее — [скачать](https://nodejs.org/)
- Доступ к Hikvision устройствам по локальной сети
- Доступ к серверу HRMS через интернет

> `npm install` НЕ нужен — агент использует только встроенные модули Node.js

---

## Установка (первый раз)

### 1. Установить Node.js

Скачать с [nodejs.org](https://nodejs.org/) (кнопка "LTS").
После установки перезагрузить компьютер.

Проверить:
```cmd
node --version
```
Должно вывести `v18.x.x` или новее.

### 2. Скопировать файлы агента

Скопировать папку `agent/` с HRMS-сервера на офисный ПК.
Необходимые файлы:
```
agent/
├── relay-agent.js   ← основной скрипт
└── AGENT.bat        ← мастер настройки (запускать от администратора)
```

### 3. Запустить AGENT.bat от имени администратора

Правой кнопкой на `AGENT.bat` → "Запуск от имени администратора"

Откроется меню:
```
  N  --  New agent (add another floor/zone)
  M  --  Manage existing agent
  Q  --  Quit
```

### 4. Создать нового агента

Нажать `N`, ввести:
- **Profile name**: короткое название (например, `office1`, `floor2`)
- **Display name**: человекочитаемое имя (например, `Главный офис`)

Скрипт создаст файл `config-office1.json` и зарегистрирует агента как задачу Windows.
Агент запустится автоматически и будет стартовать при каждой загрузке компьютера.

## Конфигурация (config.json)

После создания через AGENT.bat файл выглядит так:

```json
{
  "serverUrl": "http://185.125.200.112:7272",
  "agentToken": "beebe0a45627c39f2c3b025f338e1f2f...",
  "agentName": "Главный офис",
  "pollIntervalMs": 5000,
  "deviceCheckIntervalMs": 30000,
  "hikvisionTimeoutMs": 10000,
  "logFile": "agent-office1.log",
  "logMaxSizeMb": 10
}
```

| Поле | Описание |
|------|----------|
| `serverUrl` | URL бэкенда HRMS (порт 7272, не 7474!) |
| `agentToken` | Токен из переменной `AGENT_SECRET_TOKEN` на сервере |
| `agentName` | Отображаемое имя агента в интерфейсе |
| `pollIntervalMs` | Интервал опроса команд (5000 мс = 5 сек) |
| `deviceCheckIntervalMs` | Интервал проверки статуса устройств (30 сек) |
| `hikvisionTimeoutMs` | Таймаут ISAPI вызовов к устройствам (10 сек) |
| `logFile` | Имя файла лога |
| `logMaxSizeMb` | Максимальный размер лога до ротации |

> **Важно:** `agentToken` должен совпадать с `AGENT_SECRET_TOKEN` в `.env` на сервере.
> Если поменять токен на сервере — нужно обновить все `config-*.json` файлы агентов.

## Несколько агентов (несколько офисов)

Можно запустить несколько агентов на одном ПК (например, для разных этажей или офисов):

```
agent/
├── config-floor1.json    ← профиль "floor1"
├── config-floor2.json    ← профиль "floor2"
├── config-reception.json ← профиль "reception"
├── agent-floor1.log
├── agent-floor2.log
├── agent-reception.log
└── relay-agent.js
```

Каждый профиль — отдельная задача в Windows Task Scheduler.
Добавить новый профиль: запустить `AGENT.bat` → N.

---

## Управление (AGENT.bat)

Через меню AGENT.bat можно:

| Пункт | Действие |
|-------|----------|
| **N** | Создать нового агента (новый профиль) |
| **M** | Управлять существующими |
| **S** | Start/Restart — запустить/перезапустить |
| **L** | View log — посмотреть последние 30 строк лога |
| **R** | Reinstall — переименовать агента |
| **U** | Uninstall — удалить агента |

## Ручной запуск (без AGENT.bat)

```cmd
node relay-agent.js
```

Или с конкретным конфигом:
```cmd
node relay-agent.js --config config-floor1.json
```

## Лог-файл

Агент пишет лог в `agent-{profile}.log` (или `agent.log` для профиля по умолчанию).

Пример успешной работы:
```
[INFO] HRMS Relay Agent v2.0 started. Agent: Главный офис
[INFO] Polling http://185.125.200.112:7272/api/agent/hik-commands every 5s
[INFO] CMD #42 GRANT employee=15 device=192.168.0.160 → success
[INFO] CMD #43 REVOKE employee=7 device=192.168.0.161 → success
```

Ошибки подключения к серверу:
```
[WARN] Server unreachable: connect ECONNREFUSED — will retry in 5s
```

## Что делает агент с устройствами

### GRANT (выдача доступа)
1. Скачивает фото сотрудника с сервера (`GET /api/agent/photo/:employeeId`)
2. Создаёт пользователя на Hikvision устройстве через ISAPI
3. Загружает фото на устройство
4. Устройство добавляет лицо в базу Face ID

### REVOKE (отзыв доступа)
1. Находит пользователя на устройстве по `employeeId` (используется как `employeeNo`)
2. Удаляет пользователя и его лицо с устройства через ISAPI

### Мониторинг устройств
Каждые 30 секунд агент проверяет доступность всех известных устройств (HTTP пинг).
Если устройство ушло в офлайн или вернулось — сервер получает уведомление.

## Если агент не запускается

**Проблема**: `Node.js not found`
**Решение**: Установить Node.js 18 LTS и перезагрузить компьютер

**Проблема**: `config.json не найден`
**Решение**: Создать через AGENT.bat → N, или скопировать `config.example.json` → `config.json`

**Проблема**: `401 Unauthorized` в логах
**Решение**: Проверить что `agentToken` в `config.json` совпадает с `AGENT_SECRET_TOKEN` в `.env` на сервере

**Проблема**: Команды висят в статусе "pending", устройство не реагирует
**Решение**: Проверить что агент запущен (`AGENT.bat` → M → L → View log), проверить IP устройства

**Проблема**: Агент создан через AGENT.bat, показывается как "Running", но в разделе СКУД → Двери не появляется
**Решение**: Агент не смог подключиться к серверу при старте. Запустить вручную в CMD для диагностики:
```cmd
node relay-agent.js --config config-ИМЯ.json
```
В консоли сразу будет видна ошибка:
- `ECONNREFUSED` / `ETIMEDOUT` — роутер офиса блокирует исходящие соединения на порт 7272. Попросить IT добавить исключение в файрволе, или временно проверить с мобильного интернета
- `401 Unauthorized` — неверный `agentToken` в config.json
- `[OK] Registered as "ИМЯ"` — агент подключился, проблема была временной, теперь работает

> Раздел "Агенты" в системе (СКУД → Двери) показывает агентов только суперадмину холдинга.

## Настройка новых Hikvision устройств в HRMS

После физической установки нового терминала:

1. Настроить на устройстве **HTTP Webhook**:
   - URL: `http://SERVER_IP:7474/api/hikvision/event?token=HIKVISION_WEBHOOK_TOKEN`
   - Format: JSON
2. Поднести к устройству любое лицо → оно автоматически появится в разделе **СКУД → Устройства**
3. Привязать устройство к компании → кнопка "Привязать к компании"
4. После привязки можно выдавать доступы сотрудникам

> При первом webhook событии устройство регистрируется автоматически (модель `HikvisionDevice`).
