# HRMS Relay Agent v1.0

Локальный агент для интеграции Hikvision Face ID с центральным HRMS сервером.
Устанавливается на компьютер **в каждом офисе** (или на любую машину в той же сети что и Hikvision устройства).

**Возможности:**
- Выполняет команды выдачи/отзыва доступа на Hikvision через ISAPI
- Загружает фото лица сотрудника на устройство
- Мониторит доступность устройств каждые 30 сек
- Отправляет Telegram алерты при офлайн/онлайн устройства
- Автоматически перезапускается при сбое
- Работает как системная служба Windows (HIGH priority)

## Как это работает

```
┌─────────────────────────────────────────────────────────────┐
│                    ЦЕНТРАЛЬНЫЙ СЕРВЕР                        │
│                   (в любом месте/облаке)                     │
│                                                              │
│  Кадровик нажимает "Открыть доступ" →                       │
│  Команда сохраняется в БД со статусом "pending"              │
└───────────────────────┬─────────────────────────────────────┘
                        │ HTTPS (исходящий, порт 443)
                        │ Агент сам опрашивает сервер
                        ▼
┌─────────────────────────────────────────────────────────────┐
│              ОФИС (локальная сеть)                           │
│                                                              │
│  ┌──────────────────┐    HTTP (192.168.x.x)                 │
│  │  relay-agent.js  │ ──────────────────────▶ [Hikvision IN] │
│  │  (этот скрипт)   │ ──────────────────────▶ [Hikvision OUT]│
│  └──────────────────┘                                       │
│                                                              │
│  Агент: каждые 5 сек спрашивает "есть команды?"             │
│         выполняет ISAPI на устройства                        │
│         отчитывается серверу "выполнено/ошибка"              │
└─────────────────────────────────────────────────────────────┘
```

**Устройства никогда не торчат в интернет.**
Агенту нужен только **исходящий** интернет-доступ к серверу.

## Требования

- Windows 10/11 или Windows Server 2016+
- Node.js 18+ ([nodejs.org](https://nodejs.org))
- Доступ к интернету (для связи с HRMS сервером)
- Нахождение в той же локальной сети что и Hikvision устройства

## Установка — шаг за шагом

### Шаг 1: Скопируй папку `agent/` на компьютер в офисе

Например в `C:\hrms-agent\`

### Шаг 2: Получи токен с сервера

На сервере открой файл `backend/.env` и найди строку:
```
AGENT_SECRET_TOKEN=xxxxxxxxxxxxx
```
Это и есть твой токен.

### Шаг 3: Узнай ID компании

Войди в систему как суперадмин → `/admin` → найди нужную компанию → запомни её ID.

### Шаг 4: Создай config.json

Скопируй `config.example.json` → `config.json` и заполни:

```json
{
  "serverUrl": "https://твой-сервер.com",
  "agentToken": "значение_из_AGENT_SECRET_TOKEN",
  "companyId": 3,
  "pollIntervalMs": 5000
}
```

### Шаг 5: Проверь подключение

```
node test-connection.js
```

Должно показать `✅` для сервера и устройств.

### Шаг 6: Установи сервис

**Правый клик → Запуск от имени администратора** на `install.bat`

Готово! Агент:
- Запускается автоматически при старте Windows
- Перезапускается автоматически при сбое (через 1 минуту)
- Работает под учёткой SYSTEM с приоритетом HIGH
- Логирует всё в `agent.log`

## Файлы в папке

| Файл | Назначение |
|------|-----------|
| `УСТАНОВИТЬ.bat` | **Главный файл** — мастер установки, один клик |
| `relay-agent.js` | Основной скрипт агента |
| `config.json` | Твои настройки (создаётся при установке) |
| `config.example.json` | Шаблон настроек со всеми параметрами |

## Параметры config.json

| Параметр | По умолчанию | Описание |
|----------|-------------|---------|
| `serverUrl` | — | Адрес HRMS сервера (https://...) |
| `agentToken` | — | Значение AGENT_SECRET_TOKEN из backend/.env |
| `companyId` | — | ID компании этого офиса |
| `pollIntervalMs` | 5000 | Интервал опроса команд (мс) |
| `deviceCheckIntervalMs` | 30000 | Интервал проверки устройств (мс) |
| `logFile` | agent.log | Файл логов |
| `logMaxSizeMb` | 10 | Макс. размер лога до ротации |

```bat
:: Запустить вручную (для теста в консоли)

taskkill /f /im node.exe

:: Посмотреть логи

:: Удалить сервис
uninstall.bat
```

## Мониторинг устройств

Агент автоматически проверяет TCP-доступность каждого Hikvision устройства каждые **30 секунд**.

- При **первом запуске** — фиксирует начальное состояние (без уведомлений)
- При **потере связи** → Telegram: `🔴 УСТРОЙСТВО НЕДОСТУПНО`
- При **восстановлении** → Telegram: `🟢 Устройство ВОССТАНОВЛЕНО`
- Если состояние не меняется — уведомлений нет (нет спама)

## Что делать если что-то не работает

**❌ "Неверный токен"**
→ Проверь `agentToken` в config.json совпадает с `AGENT_SECRET_TOKEN` в `backend/.env` на сервере

**❌ "Сервер недоступен"**
→ Проверь `serverUrl` — правильный ли домен/IP
→ Проверь интернет на этом компьютере
→ Если сервер на HTTP (не HTTPS) — убедись что порт указан: `http://ip:7474`

**❌ "Timeout 192.168.x.x"**
→ Проверь что компьютер с агентом и Hikvision в одной сети
→ Попробуй пинговать: `ping 192.168.1.100`
→ Проверь что в системе правильно прописан IP устройства

**❌ "HTTP 401 от Hikvision"**
→ Неверный логин/пароль устройства
→ В системе `/doors` проверь поля "Логин" и "Пароль" для этой двери

**❌ Команды висят в статусе "processing" и не выполняются**
→ Агент взял команду но упал
→ Перезапусти агент — команды автоматически сбросятся в "pending"

## Настройка для нескольких офисов

В **каждом** офисе:
1. Скопируй папку `agent/`
2. Создай свой `config.json` с нужным `companyId`
3. Запусти `install.bat`

Один агент = один офис (одна компания).

---

## Безопасность

- Токен агента хранится в `config.json` — не показывай его посторонним
- Агент использует только HTTPS к серверу (или HTTP если сервер без SSL)
- Hikvision устройства доступны только изнутри локальной сети
- Пароли устройств хранятся зашифрованно на сервере, передаются агенту по HTTPS
